How Threadline Was Built

What we set out to build, what changed along the way, and why — told as a founder's account.

Chapter 1 — The original idea
Started as a PDF report generator
The first concept: a brand PM types in a product idea, Threadline validates it against consumer data, and generates a downloadable PDF report. Static. One-time. Deliverable.
↓
Pivot: PDF → live web app
A PDF gets stale the moment it's generated. Consumer conversations on Reddit keep moving. The intelligence needs to be live, not a snapshot — so we scrapped the report entirely and built a web application instead.
Chapter 2 — Rethinking the user
Before
Validation tool
PM types a product idea → Threadline scores it → brief returned. Only helps people who already have an idea.
After
Discovery tool
PM picks a condition → Threadline surfaces the ideas → PM leaves knowing what to build. No input required.
The pivot came from a simple question: what does a brand entering adaptive fashion for the first time type into a text box? Nothing. They don't have an idea yet — that's exactly why they need Threadline. So the text box was removed entirely.
Chapter 3 — Intelligence layers added
Cross-condition overlap
A front-closure garment is needed by post-mastectomy, post-surgical, and rheumatoid patients. Threadline detects this automatically and flags it — a larger market opportunity than one condition alone.
Signal strength scoring
Every opportunity gets a score (0–100) and confidence level (High / Medium / Low). Score = signal volume + consistency. Confidence = data depth. Both together tell the full story.
Pre-generation architecture
Originally, Claude would generate opportunities when a user clicked. Changed to weekly pre-generation — Opus runs once a week, saves to DB, users get instant results. Zero LLM cost at query time.
Talk with Reports (Phase 2)
After reading a brief, PMs will be able to ask questions grounded in real consumer data — answered by Claude Sonnet using the vector store as context.
Chapter 4 — Getting the data
Tried & blocked
Reddit API
App registration at reddit.com/prefs/apps failed repeatedly — a known CAPTCHA bug. Could not get credentials.
Tried & blocked
Amazon direct scraping
HTTP scraper returned zero reviews across all tested product pages. Amazon's anti-scraping infrastructure blocked every request immediately.
What we used
Hugging Face — Amazon Reviews 2023
McAuley Lab academic dataset. Streamed 500k records, filtered for adaptive keywords → 2,192 real reviews. Cleaner than direct scraping — verified purchases, community-voted quality.
What we used
Reddit RSS feed
Public RSS at reddit.com/r/subreddit/hot/.rss — no API key needed. Pulled hot posts from 10 adaptive fashion communities → 129 posts. Hot = community-validated signal.
Chapter 5 — The pipeline
1
Scrape
Reddit RSS (hot posts) + Hugging Face Amazon dataset → raw consumer text into Supabase
2
Clean
Remove HTML, salutations, noise, short posts. Added after seeing how much irrelevant content was reaching extraction. Dirty input → poor LLM output.
3
Extract — Claude Haiku
Runs on every record. Extracts pain points, features, sentiment. Haiku chosen for speed and cost at 2,000+ records/week.
4
Embed — OpenAI text-embedding-3-small
1,536-dimension vectors. Enables semantic search — find by meaning, not keywords. Full dataset cost: under $0.05.
5
Synthesise — Claude Opus 4.8
Runs once a week. Reads top signals per condition, generates ranked opportunities + full briefs, saves to DB. Users never trigger this — results are pre-built and instant.
